## 6. Git Process

Team org URL:
https://github.com/capstone-nova-minds

Repo URL:
https://github.com/capstone-nova-minds/main-repo

Project board URL:
https://github.com/orgs/capstone-nova-minds/projects/3

Branch strategy:
- The main branch is protected.
- No direct commits to main.
- All changes must be done through Pull Requests.

Pull Request review workflow:
- Each Pull Request requires 2 approvals before merging.
- Stale approvals are dismissed when new commits are pushed.
- Code Owner review is not required.

Merge cadence:
- Small changes can be merged after review.
- Large features should be split into smaller Pull Requests.
- The team will review Pull Requests daily during the sprint.

Bypass rule:
- The bypass list will remain empty.
- No teammate should bypass the branch protection rules.
